## Overview

Automations support repeatable work in Vala. Exact triggers, actions, and availability depend on the configured product experience.

## When to use

Use Automations when your organization has an approved repeatable workflow that should run consistently.

## Prerequisites

- Administrative or workflow access as required
- A clearly defined process
- Approval from the appropriate organization owner

## Best practices

- Start with a narrow, reviewable scope.
- Keep a human review step for consequential outputs.
- Recheck automations when your process changes.

## Common mistakes

- Enabling an untested workflow broadly
- Using unclear triggering criteria
- Failing to review automated results
